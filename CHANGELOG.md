# Changelog

## [0.2.1] - 2026-04-08

### Changed
- Improved dirty-files prompt to `How to proceed? [c]ommit staged / [y] continue / [n] abort` for clarity.

## [0.2.0] - 2026-04-08

### Added
- **Interactive commit flow for staged files** — the dirty-files prompt now includes a `[c]ommit staged` option when staged files are present.
  - Staged, unstaged, and untracked files are displayed in separate color-coded tables (green / yellow / dim) instead of a single combined table.
  - Numbered multi-select lets you pick a subset of staged files to include in the commit; unselected staged files remain staged.
  - Commit message can be typed manually or generated by the configured LLM backend.
  - LLM-generated messages can be accepted, opened in `$EDITOR` for editing, or rejected in favor of a manually typed message.
  - Non-interactive (`--yes`) mode preserves the existing behavior: display only, no prompts.

## [0.1.0] - 2024-01-01

### Added
- Initial release: automated PR preparation workflow combining branch safety checks, pre-commit hooks, git rebase, LLM-generated PR title and body, Jira linking, and GitHub PR creation/update.
- Stat-only and full-diff (`--full-diff`) multi-agent LLM generation modes.
- Layered YAML configuration (`~/.prflow.yaml` + `.prflow.yaml`).
- Pluggable LLM backends: Claude CLI, OpenAI (stub), custom command.
- Pluggable Jira backends: URL-only, REST API (stub), MCP (stub).
- `--dry-run`, `--yes`, `--seed`, `--draft`, `--base`, `--no-rebase`, `--no-pre-commit` flags.
